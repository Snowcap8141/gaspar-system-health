# Security Policy

## Supported Version

Current maintained release line:

- `v1.0.7`

## Reporting a Security Issue

If a security issue is identified in Gaspar System Health:

1. avoid immediate public disclosure of exploit details
2. describe the issue clearly
3. include affected version, impact, and reproduction details when available
4. attach logs or screenshots only when they do not expose sensitive data

## Security Notes

- the application is intended for administrative diagnostics on Windows
- releases should be obtained only from the official project repository or release page
- integrity verification through published SHA256 hashes is recommended
- public review should compare the release archive with the corresponding source version

## Authenticity and Integrity

Public releases should be verified through:

- repository provenance
- release package consistency
- published SHA256 hashes
- matching source code for the declared release version
